Managers
========

GameManager (NetworkBehaviourSingleton<GameManager>)
-----------------------------------------------------
The central game state manager. Synchronized across all clients.

  GameManager.Instance.GameState   // NetworkVariable<GameState>
  GameManager.Instance.Phase       // GamePhase (convenience from GameState)
  GameManager.Instance.Tick        // int (current game tick)
  GameManager.Instance.Period      // int (current period, 1-indexed)
  GameManager.Instance.BlueScore   // int
  GameManager.Instance.RedScore    // int
  GameManager.Instance.IsOvertime  // bool

GameState
---------
  struct GameState
  {
      GamePhase Phase;
      int Tick;
      int Period;
      int BlueScore;
      int RedScore;
      bool IsOvertime;
  }

Listen for changes via:
  GameManager.Instance.GameState.OnValueChanged += (old, newVal) =>
  {
      Debug.Log($"Phase: {newVal.Phase}, Score: {newVal.BlueScore}-{newVal.RedScore}");
  };

Or via the event system:
  EventManager.AddEventListener("Event_Everyone_OnGameStateChanged", handler);

PlayerManager (MonoBehaviourSingleton<PlayerManager>)
-----------------------------------------------------
Manages all Player objects in the game. Use this for player lookups and queries.

  PlayerManager.Instance.GetPlayers(bool includeReplay = false)
  PlayerManager.Instance.GetPlayersByPhase(PlayerPhase phase)
  PlayerManager.Instance.GetPlayersByTeam(PlayerTeam team)
  PlayerManager.Instance.GetPlayersByTeams(PlayerTeam[] teams)
  PlayerManager.Instance.GetPlayerByClientId(ulong clientId)
  PlayerManager.Instance.GetPlayerByUsername(FixedString32Bytes username)
  PlayerManager.Instance.GetPlayerByNumber(int number)
  PlayerManager.Instance.GetPlayerBySteamId(FixedString32Bytes steamId)
  PlayerManager.Instance.GetPlayerByNeedle(string needle)
  PlayerManager.Instance.GetLocalPlayer()
  PlayerManager.Instance.GetSpawnedPlayers(bool includeReplay = false)
  PlayerManager.Instance.GetSpawnedPlayersByTeam(PlayerTeam team)
  PlayerManager.Instance.GetReplayPlayers()

GetPlayerByNeedle() accepts partial matches against username or player number,
useful for implementing chat commands like "/msg playerName".

GetLocalPlayer() is the most common accessor - it returns the local user's
Player instance. On a dedicated server, this returns null (no local player).

PuckManager (MonoBehaviourSingleton<PuckManager>)
--------------------------------------------------
  PuckManager.Instance    // Singleton instance

  // Queries
  puckManager.GetPucks(bool includeReplay = false)   // List<Puck>
  puckManager.GetPuck(bool includeReplay = false)    // First non-replay puck
  puckManager.GetReplayPucks()                       // List<Puck>
  puckManager.GetPuckByNetworkObjectId(ulong id)     // Puck by NetworkObjectId
  puckManager.GetReplayPuckByNetworkObjectId(ulong id)
  puckManager.GetPlayerPuck(ulong clientId)          // Puck currently touching player's stick
  puckManager.AddPuckPosition(PuckPosition puckPos)
  puckManager.RemovePuckPosition(PuckPosition puckPos)

  // Server-only
  puckManager.Server_SpawnPuck(Vector3 pos, Quaternion rot, bool isReplay = false)
  puckManager.Server_DespawnPuck(Puck puck)
  puckManager.Server_DespawnPucks(bool includeReplay = false)
  puckManager.Server_SpawnPucksForPhase(GamePhase phase)  // spawns for each PuckPosition

GetPlayerPuck() returns the first puck in the collision buffer of the player's stick,
useful for "who has the puck" queries or pass detection.

ChatManager (NetworkBehaviourSingleton<ChatManager>)
-----------------------------------------------------
  ChatManager.Instance    // Singleton instance

  // Client methods
  chatManager.Client_SendChatMessage(string content, bool isQuickChat, bool isTeamChat)
  chatManager.Client_QuickChatAction(int index)
  chatManager.SetQuickChatEnabled(bool enabled, QuickChatCategory? category = null)

  // Server methods
  chatManager.Server_SendChatMessage(ChatMessage msg, params ulong[] clientIds)
  chatManager.Server_SendChatMessage(string content, string color, params ulong[] clientIds)
  chatManager.Server_BroadcastChatMessage(ChatMessage msg)
  chatManager.Server_BroadcastChatMessage(string content, string color = null)

  // Local-only
  chatManager.AddChatMessage(ChatMessage msg)
  chatManager.RemoveChatMessage(ChatMessage msg)
  chatManager.ClearChatMessages()
  chatManager.ParseChatMessageContent(string content)  // trims/clamps to max length

Server methods send messages over the network via RPCs. The string+color overload
creates a system message wrapped in a rich text color tag (e.g. "#b8b8b8", "#e74c3c").
Broadcast variants send to every connected client; the params overload targets
specific client IDs.

InputManager
------------

rebind keys.

  InputManager.StopAction
  InputManager.TwistLeftAction
  InputManager.TwistRightAction
  InputManager.DashLeftAction
  InputManager.DashRightAction
  InputManager.ExtendLeftAction
  InputManager.ExtendRightAction

  // Quick Chat

Each action exposes a ReadValue<T>() method to get current input state:
  Vector2 stickInput = InputManager.StickAction.ReadValue<Vector2>();
  bool isSprinting = InputManager.SprintAction.ReadValue<float>() > 0.5f;

SaveManager
-----------
Wraps Unity's PlayerPrefs. Use this for persisting mod configuration.

  SaveManager.SetBool(string key, bool value)
  SaveManager.GetBool(string key, bool defaultValue)
  SaveManager.SetInt(string key, int value)
  SaveManager.GetInt(string key, int defaultValue)
  SaveManager.SetFloat(string key, float value)
  SaveManager.GetFloat(string key, float defaultValue)
  SaveManager.SetString(string key, string value)
  SaveManager.GetString(string key, string defaultValue)
  SaveManager.SetEnum<T>(string key, T value)
  SaveManager.GetEnum<T>(string key, T defaultValue)

Best practice: Prefix your keys with a unique mod identifier:
  SaveManager.SetFloat("myMod_volume", 0.75f);
  float vol = SaveManager.GetFloat("myMod_volume", 0.75f);

Item
----
  item.id
  item.name
  item.description
  item.categories    // string[] - "flag", "headgear", "jersey", "stickSkin", etc.
  item.price
  item.code

Items are cosmetic unlocks (flags, jerseys, stick skins, etc.). Categories
determine which slot an item equips to. Use ItemManager.GetItemsByCategories()
to filter by type.

ConnectionManager (MonoBehaviourSingleton<ConnectionManager>)
-------------------------------------------------------------
Manages client connections to remote servers.

  // Properties
  connectionManager.UnityTransport  // UnityTransport component

  // Client methods
  connectionManager.Client_StartClient(string ipAddress, ushort port, string password = null)

Client_StartClient initiates a connection to the given IP and port. If already
connected to another server, it disconnects first. The password is optional and
sent alongside the player's ConnectionData.

ServerManager (NetworkBehaviourSingleton<ServerManager>)
---------------------------------------------------------
The host/server manager. Controls server lifecycle, manages admin/ban/whitelist
functionality, and provides port forwarding support.

  // Sub-managers
  serverManager.EdgegapManager              // EdgegapManager (dedicated server orchestration)
  serverManager.ConnectionApprovalManager   // ConnectionApprovalManager
  serverManager.TimeoutManager              // TimeoutManager
  serverManager.BanManager                  // BanManager
  serverManager.AdminManager                // AdminManager
  serverManager.WhitelistManager            // WhitelistManager
  serverManager.ServerConfig                // ServerConfig
  serverManager.TcpServer                   // TCPServer (server browser preview)
  serverManager.IpAddress
  serverManager.IsHostStartInProgress

  // Server lifecycle
  serverManager.StartHost(ushort port, string name, int maxPlayers, string password, bool isPublic, bool useVoip, bool forwardPorts = false)
  serverManager.StartServer(ushort port, bool forwardPorts = false)
  serverManager.StartTcpServer(ushort port)       // LAN server browser discovery
  serverManager.StopTcpServer()
  serverManager.HandleStartFailure()              // Called on port conflict
  serverManager.Authenticate()                    // Registers with Puck backend
  serverManager.Unauthenticate()                  // Unregisters from Puck backend
  serverManager.StartPortForwarding(ushort port)
  serverManager.StopPortForwarding()

  // Moderation
  serverManager.Server_KickPlayer(Player player, DisconnectionCode code = Kicked, string message = null, bool applyTimeout = true)
  serverManager.Server_BanPlayer(Player player)
  serverManager.Server_BanSteamId(string steamId)
  serverManager.Server_UnbanSteamId(string steamId)
  serverManager.Server_IsSteamIdBanned(string steamId)

StartHost creates a listen server and also joins as a client (for local play).
StartServer creates a dedicated server instance. TCP server is optional and used
for LAN server browser discovery so other clients can see server name/player count.

VoteManager
-----------
  VoteManager.Instance  // Singleton

Manages votes (forfeit votes, etc.). Handled automatically by BaseGameMode but
available for custom game modes to inspect or trigger votes.

Vote
----
  Vote is a class passed through vote-related events:
    vote.Title        // string - e.g. "Forfeit"
    vote.Description  // string
    vote.SteamId      // string - vote creator's Steam ID
    vote.Teams        // PlayerTeam[] - which teams participate
    vote.InFavourVotes
    vote.RequiredVotes
    vote.Passed       // bool
